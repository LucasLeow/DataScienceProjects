Data Preparation

Data Source

Data was stored in 5 separate csv files.
  1. Resale Flat Data from 1990 - 1999 (287,196 entries)
  2. Resale Flat Data from 2000 - 2012 (369,651 entries)
  3. Resale Flat Data from 2012 - 2014 (52,203 entries)
  4. Resale Flat Data from 2015 - 2016 (37,153 entries)
  5. Resale Flat Data from 2017 onwards (130,404 entries)

Handling Missing Values

Owing to large number of entries, only resale flat data from 2012 onwards was used to speed up data processing.
This reduces the number of entries from 876,607 to 219,760. Moreover, resale prices from earlier years are much lower compared to current market prices.
The usage of older data will be covered in future sections, for the purpose of pattern / trend discovery.

Identifying Missing Data


Only the column "remaining_lease" has null values. Singapore HDB flats are sold on 99 year leases.
Hence, the missing 'remaining lease' values can be obtained by taking : 99 - (Registration Year - Lease Commence Date Year)

Discovering non-standardized format


Not all values in the 'remaining_lease' column are in standard format. Some were in integer values and some were written in years and months format as seen in the image.
Hence, the existing values have to be first standardized before the null values can be handled.

Problem Definition and Feature Selection

After cleaning the data, it is time to proceed with data analysis. It is important to first define the problem to be solved. For this project, the goal is to find out factors affecting resale value of HDB flats and thereby, predict future resale prices based on these factors.

There are 11 features and 1 label.

    Features selected are:
  1. town
  2. flat_type
  3. storey_range
  4. floor_area_sqm
  5. flat_model
  6. remaining_lease
Label : Resale Price
